### Key palindrome fact
* A palindrome is symmetric.
* So for each character, you can use its letters in pairs (one on left, one on right).
* If any character has an odd count, you can place exactly one leftover character in the center.
- ```s``` contains lowercase and/or uppercase English letters (52 chars), but implementations often choose 128 anyway because:

    <li>No mapping needed: you can do cnt[c]++ directly.</li>
    <li>Safer with unexpected input: if there are other ASCII characters, it still works.t</li>

Better:

```cpp
#include <string>
#include <array>
using namespace std;

class Solution {
public:
    int longestPalindrome(string s) {
        array<int, 128> cnt{};          // ASCII counts
        for (unsigned char c : s) cnt[c]++;

        int len = 0;
        bool hasOdd = false;

        for (int f : cnt) {
            len += (f / 2) * 2;         // take all pairs
            if (f % 2 == 1) hasOdd = true;
        }

        if (hasOdd) len += 1;           // one odd char can sit in the center
        return len;
    }
};
```
This solves LeetCode 409 (**Longest Palindrome**): given a string `s`, return the **maximum length** of a palindrome you can build using its letters (you can reorder letters).

### Key palindrome fact
- A palindrome is symmetric.
- So for each character, you can use its letters in **pairs** (one on left, one on right).
- If any character has an **odd** count, you can place **exactly one** leftover character in the **center**.

---

## Walkthrough of the code

### 1) Count frequency of every ASCII character
```cpp
array<int, 128> cnt{};
for (unsigned char c : s) cnt[c]++;
```
- `cnt[x]` stores how many times the character with ASCII code `x` appears.
- Using size `128` lets us directly index by the character code (handles `A-Z`, `a-z`, etc.).
- `unsigned char` prevents negative indexing if `char` is signed on your compiler.

---

### 2) Build the palindrome length from counts
```cpp
int len = 0;
bool hasOdd = false;

for (int f : cnt) {
    len += (f / 2) * 2;
    if (f % 2 == 1) hasOdd = true;
}
```

For each frequency `f`:
- `(f / 2)` = number of pairs available
- `(f / 2) * 2` = how many characters you can use in pairs (always even)

Examples:
- `f = 7` → pairs = `7/2 = 3` → use `3*2 = 6` characters, leftover `1`
- `f = 4` → pairs = `2` → use `4`, leftover `0`
- `f = 1` → pairs = `0` → use `0`, leftover `1`

`hasOdd` becomes `true` if **any** character had an odd frequency—meaning there exists at least one leftover character that could go in the center.

---

### 3) Add one center character if possible
```cpp
if (hasOdd) len += 1;
return len;
```
- You can put **only one** odd leftover character in the middle of the palindrome.
- Even if multiple characters are odd, you still add just **1** total for the center.

---

## Quick example
`s = "abccccdd"`
- counts: `a1, b1, c4, d2`
- pairs used: `a0 + b0 + c4 + d2 = 6`
- there are odd counts (`a` and `b`) → `hasOdd = true`
- add center `+1` → total `7`

So answer is `7` (e.g., `"dccaccd"`).

## Complexity
- Time: **O(n + 128)** ≈ **O(n)**
- Space: **O(128)** ≈ constant

If you want, I can show how to compute the actual palindrome string too (not just the length).
You can construct one valid longest palindrome by:
1) count frequencies
2) put `count/2` copies of each char into a `left` string
3) pick one odd-count char as the `center` (if any)
4) `right` is the reverse of `left`
5) answer = `left + center + right`

Here’s C++ code that returns the palindrome string (ASCII-based, like your 128 solution):

```cpp name=build_longest_palindrome.cpp
#include <string>
#include <array>
#include <algorithm>
using namespace std;

class Solution {
public:
    // Builds one of the longest palindromes possible from s
    string buildLongestPalindrome(string s) {
        array<int, 128> cnt{};
        for (unsigned char c : s) cnt[c]++;

        string left;
        left.reserve(s.size() / 2);

        char center = 0;        // 0 means "no center chosen"
        bool hasCenter = false;

        for (int ch = 0; ch < 128; ++ch) {
            int pairs = cnt[ch] / 2;               // number of pairs
            left.append(pairs, (char)ch);          // put one side of each pair in left

            if (!hasCenter && (cnt[ch] % 2 == 1)) {
                center = (char)ch;                 // pick any odd char as center
                hasCenter = true;
            }
        }

        string right = left;
        reverse(right.begin(), right.end());

        if (hasCenter) return left + string(1, center) + right;
        return left + right;
    }
};
```

Notes:
- This builds a palindrome of **maximum length**, but not necessarily unique.
- If you want the palindrome to use letters in a specific order (e.g., lexicographically smallest), we can adjust how we choose `center` and how we build `left`.
