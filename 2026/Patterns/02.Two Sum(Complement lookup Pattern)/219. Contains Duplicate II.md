# Right Bruteforce
```cpp
class Solution {
public:
    bool containsNearbyDuplicate(vector<int>& nums, int k) {
        for (int i = 0; i < (int)nums.size(); ++i){
            for (int j = i+1;j < (int)nums.size(); ++j){
                if (nums[i] == nums[j] && abs(i-j) <= k)
                    return true;
            }
        }
        return false;
    }
};
```
# Optimize 1 step
```cpp
class Solution {
public:
    bool containsNearbyDuplicate(vector<int>& nums, int k) {
        for (int i = 0; i < (int)nums.size(); ++i){
            for (int j = i+1;j < (int)nums.size(); ++j){
                if (j - i > k) break; //optimization
                if (nums[i] == nums[j] && abs(i-j) <= k)
                    return true;
            }
        }
        return false;
    }
};
```
# Sliding Window Optimized
```cpp
class Solution {
public:
    bool containsNearbyDuplicate(vector<int>& nums, int k) {
        unordered_set<int> window;

        for (int i = 0; i < (int)nums.size(); ++i) {
            if (window.count(nums[i])) return true;

            window.insert(nums[i]);

            // keep only last k elements in the set
            if ((int)window.size() > k) {
                window.erase(nums[i - k]);
            }
        }
        return false;
    }
};
```
This logic implements a **sliding window of size** ```k``` over the array and uses a hash set to detect duplicates **only within that window.**

## What window represents
* ```window``` is an ```unordered_set<int>``` holding the values from the most recent indices near i.
* At the start of each loop iteration for index ```i```, ```window``` is intended to contain values from indices:
```[i−k, i−1]```
(i.e., the previous up-to-```k``` elements).
If a value repeats inside this range, then there exist indices ```p < i``` such that:

```nums[p] == nums[i]```
```i - p <= k```
That’s exactly the condition the problem asks for.

Step-by-step through the loop
## 1) ```if (window.count(nums[i])) return true;```
* ```window.count(x)``` is ```1``` if ```x``` is already in the set, else ```0```.
* If ```nums[i]``` is already in ```window```, that means the same value occurred at some recent index ```p``` within the last ```k``` steps.
* Therefore ```abs(i - p) <= k``` (actually ```i - p <= k``` since ```p < i```), so we can immediately return ```true```.
## 2) ```window.insert(nums[i]);```
* Add the current value so that upcoming elements can detect duplicates with it.
## 3) Maintain the sliding window size
```cpp
if ((int)window.size() > k) {
    window.erase(nums[i - k]);
}
```
This part ensures we don’t keep elements that are more than k indices behind.

* After inserting nums[i], the set might now represent indices roughly [i-k, i] (size up to k+1).
* If it grows beyond k, we remove the element that is exactly k positions behind the current index: nums[i - k].
Why nums[i - k]?

* When you move from index i to i+1, the valid “previous indices” range becomes:
```[(i+1)−k, i]=[i−k+1, i]```
* So index i-k falls out of the allowed distance and must be removed.
* nums[i-k] is exactly the value at that expiring index.
<hr>
### Small example (to visualize)
```nums = [1, 2, 3, 1], k = 3```

* ```i=0```, window {} → not found → insert 1 → window {1}
* ```i=1```, window {1} → 2 not found → insert 2 → {1,2}
* ```i=2```, window {1,2} → 3 not found → insert 3 → {1,2,3}
* ```i=3```, window {1,2,3} → 1 found → return true Distance is 3 - 0 = 3 <= k.
<hr>
### Important subtlety: why set size works here
The code uses window.size() > k as the trigger. Conceptually we want to keep at most k previous elements (indices [i-k, i-1]).

* After insertion, we temporarily have up to k+1 elements (previous k plus current).
* Removing nums[i-k] brings it back in sync for the next iteration.
Edge case:

* If k == 0, then window.size() > 0 becomes true right after inserting, and it erases nums[i] immediately. So the window is always empty at the next iteration, and duplicates are never detected—correct, because distance must be 0 (which would require i==j, disallowed).
<hr>
### Complexity
* Average time: O(n) (each insert/erase/lookup is average O(1))
* Space: O(min(n, k)) (window holds at most k elements)
