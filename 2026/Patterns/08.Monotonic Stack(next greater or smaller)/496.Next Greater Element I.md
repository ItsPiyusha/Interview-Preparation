### 496 — Next Greater Element I (C++): monotonic stack pattern (step-by-step)

**Problem in one line:**  
For each value in `nums1`, find the **first greater element to its right** in `nums2` (or `-1` if none).

This is the classic **Next Greater Element (NGE)** pattern.

---

## Pattern to learn: “Monotonic decreasing stack + map answers”

### Why a stack?
When scanning `nums2` left → right, whenever you find a number `x`, it can become the **next greater** for some previous smaller numbers. The stack helps track “numbers still waiting to find their next greater”.

### Monotonic stack invariant
We maintain a stack such that values are **strictly decreasing from bottom to top**.

That means:
- Stack top is the **most recent** number that hasn’t found a greater element yet.
- If current value `cur` is greater than stack top, then `cur` is the next greater for that top (and maybe for more elements below it).

---

## Step-by-step algorithm

### Step 1: Build next-greater mapping for every element in `nums2`
We compute `nextGreater[val] = next greater value` using a stack.

Initialize:
- `stack<int> st;`  // holds values from nums2 (or indices; values are fine here because nums2 has unique elements in this problem)
- `unordered_map<int,int> nge;`

Now iterate through `nums2`:
1. Let `cur = nums2[i]`
2. While stack not empty AND `cur > st.top()`:
   - The top element `smaller = st.top()` has found its next greater: `cur`
   - `nge[smaller] = cur`
   - Pop it (it’s resolved)
3. Push `cur` onto the stack (it is now waiting for a future greater element)

After the loop:
- Anything still in the stack has **no greater element to the right**
- Set `nge[value] = -1` for those remaining

### Step 2: Answer queries from nums1
For each `x` in `nums1`, the answer is `nge[x]`.

---

## Walkthrough example (important to internalize)

`nums2 = [1, 3, 4, 2]`

We scan:

- cur=1: stack=[], push 1 → stack=[1]
- cur=3: 3 > 1 → nge[1]=3, pop 1; push 3 → stack=[3]
- cur=4: 4 > 3 → nge[3]=4, pop 3; push 4 → stack=[4]
- cur=2: 2 is not > 4; push 2 → stack=[4,2]

End: remaining stack values have nge = -1:
- nge[2]=-1
- nge[4]=-1

So mapping:
- 1→3, 3→4, 4→-1, 2→-1

Then for `nums1 = [4,1,2]`:
- 4→-1, 1→3, 2→-1

## C++ implementation (clean and standard)

```cpp name=next_greater_element_i.cpp
#include <bits/stdc++.h>
using namespace std;

class Solution {
public:
    vector<int> nextGreaterElement(vector<int>& nums1, vector<int>& nums2) {
        unordered_map<int,int> nge;   // value -> next greater value
        stack<int> st;                // monotonic decreasing stack of values

        // Step 1: preprocess nums2
        for (int cur : nums2) {
            while (!st.empty() && cur > st.top()) {
                nge[st.top()] = cur;
                st.pop();
            }
            st.push(cur);
        }

        // remaining have no next greater
        while (!st.empty()) {
            nge[st.top()] = -1;
            st.pop();
        }

        // Step 2: answer for nums1
        vector<int> ans;
        ans.reserve(nums1.size());
        for (int x : nums1) {
            ans.push_back(nge[x]);
        }
        return ans;
    }
};
```

## What to “learn” as a reusable monotonic-stack template

### When you hear:
- “next greater element”
- “next smaller element”
- “first greater to the right”
- “days until warmer”
- “nearest smaller to left/right”

### You decide:
1. **Direction:** usually left → right for “next to the right”
2. **Stack stores:** indices (most common) or values (works here due to uniqueness)
3. **Monotonic type:**
   - Next **greater** to right → stack is **decreasing**
   - Next **smaller** to right → stack is **increasing**
4. **Pop condition:**
   - For next greater: `while cur > stackTop`
   - For next smaller: `while cur < stackTop`

---

## Complexity
- Time: **O(n)** preprocessing + **O(m)** answering, total **O(n + m)**
- Space: **O(n)** for stack + map
