### How to recognize Flood Fill (BFS shortest-path pattern family)
You should think **“flood fill BFS/DFS”** when you see:

- A **grid** (matrix) and you can move in **4 directions** (up/down/left/right) (sometimes 8).
- You’re asked to change/collect all cells in the **same connected region** starting from a given cell.
- The rule is: “You can go to neighbor if it matches some condition” (here: same original color).
- You’re **not** asked for a minimum distance necessarily, but it’s the **same BFS machinery** as shortest path in unweighted grid (queue + visited).  
  (BFS is “level order”; flood fill is “visit all reachable”.)

This is basically: **Connected Component from a starting node in a grid graph**.

---

## Step-by-step pattern to learn (BFS template)

### Step 1: Model the problem as a graph
- Each cell `(r, c)` is a node.
- There is an edge from `(r, c)` to its neighbors `(r±1, c)` and `(r, c±1)` if inside bounds.
- We only traverse nodes that have the **starting color**.

### Step 2: Decide what is “visited”
To avoid infinite loops, once you process a cell, mark it visited.
In Flood Fill, easiest: **recolor the cell** to `newColor` and treat that as visited.

### Step 3: BFS core steps
1. Push the start cell into a queue.
2. While queue not empty:
   - Pop a cell.
   - For each neighbor:
     - If in bounds and neighbor color == original color:
       - Recolor neighbor
       - Push neighbor

### Step 4: Critical edge case
If `newColor == originalColor`, then you would “recolor” to the same value and never prevent revisits (or you do extra work).  
So do an early return.

## C++ BFS solution (clean + standard)

```cpp name=flood_fill_bfs.cpp
#include <bits/stdc++.h>
using namespace std;

class Solution {
public:
    vector<vector<int>> floodFill(vector<vector<int>>& image, int sr, int sc, int newColor) {
        int m = (int)image.size();
        int n = (int)image[0].size();

        int original = image[sr][sc];
        if (original == newColor) return image; // important edge case

        queue<pair<int,int>> q;
        q.push({sr, sc});
        image[sr][sc] = newColor; // mark visited by recoloring

        int dr[4] = {-1, 1, 0, 0};
        int dc[4] = {0, 0, -1, 1};

        while (!q.empty()) {
            auto [r, c] = q.front();
            q.pop();

            for (int k = 0; k < 4; k++) {
                int nr = r + dr[k];
                int nc = c + dc[k];

                if (nr < 0 || nr >= m || nc < 0 || nc >= n) continue;
                if (image[nr][nc] != original) continue;

                image[nr][nc] = newColor; // recolor = visited
                q.push({nr, nc});
            }
        }
        return image;
    }
};
```

## “Pattern checkpoints” (how to train your brain)

When you read a grid BFS problem, ask these in order:

1) **Start node(s)?**
- Here: exactly one start `(sr, sc)`.

2) **Neighbors?**
- Usually 4-direction grid.
- Some problems allow 8-direction or knight moves, etc.

3) **Allowed move condition?**
- Here: neighbor must have `sameColor == original`.

4) **Visited strategy?**
- Option A: separate `visited[m][n]`
- Option B: modify grid in-place (here, recolor)

5) **What’s the output?**
- Here: modified grid after visiting all reachable cells.

If a problem asks “minimum steps/time” in an unweighted grid, it’s the same checklist but you also track **distance/levels**.

---

## Why BFS works here (even though shortest path isn’t asked)
Flood fill doesn’t need shortest distance, but BFS guarantees:
- Every reachable valid cell is visited once.
- Time is linear in number of visited cells.

DFS would also work; BFS is just a consistent “queue template” you can reuse for shortest path problems.

---

## Complexity
- **Time:** `O(m*n)` worst-case (if entire grid is one region).
- **Space:** `O(m*n)` worst-case queue.

---

## Common mistakes to avoid
1) Forgetting `if (original == newColor) return image;`
2) Not marking visited early (mark when you push, not when you pop) → can enqueue duplicates.
3) Mixing up bounds `m` vs `n`.
4) Using `image[nr][nc] == newColor` as visited check (wrong) without the early return.
