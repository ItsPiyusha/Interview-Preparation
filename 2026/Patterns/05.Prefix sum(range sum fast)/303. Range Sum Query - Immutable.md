### 303. Range Sum Query – Immutable (C++) — detailed explanation + the reusable pattern

The standard solution is: **precompute prefix sums once**, then answer each `sumRange(l, r)` in **O(1)**.

---

## Core idea (math)
Let `pref[i]` store the sum of the first `i` elements:

- `pref[0] = 0`
- `pref[1] = nums[0]`
- `pref[2] = nums[0] + nums[1]`
- …
- `pref[i] = nums[0] + ... + nums[i-1]`

Then the sum from index `l` to `r` is:

\[
sum(l..r) = pref[r+1] - pref[l]
\]

Why? Because `pref[r+1]` includes sum up to `r`, and subtracting `pref[l]` removes everything before `l`.

---

## C++ code (typical LeetCode class)

```cpp name=NumArray.cpp
#include <vector>
using namespace std;

class NumArray {
private:
    vector<int> pref; // pref[i] = sum of nums[0..i-1]

public:
    NumArray(vector<int>& nums) {
        int n = (int)nums.size();
        pref.resize(n + 1);
        pref[0] = 0;

        for (int i = 0; i < n; i++) {
            pref[i + 1] = pref[i] + nums[i];
        }
    }

    int sumRange(int left, int right) {
        return pref[right + 1] - pref[left];
    }
};
```

---

## Line-by-line explanation

### Data member
- `vector<int> pref;`
  - We store prefix sums.
  - Size is `n + 1` so that we can keep `pref[0] = 0` and avoid edge cases when `left = 0`.

### Constructor: `NumArray(nums)`
This is the “preprocessing” step.
- `pref.resize(n + 1);`
  - If `nums` has `n` elements, `pref` has indices `0..n`.
- `pref[0] = 0;`
  - Sum of zero elements is 0.
- Loop:
  - `pref[i + 1] = pref[i] + nums[i];`
  - Example: if `i=0`, `pref[1] = pref[0] + nums[0]` → sum of first 1 element.
  - if `i=1`, `pref[2] = pref[1] + nums[1]` → sum of first 2 elements, etc.

After this, prefix sums are ready.

### Query: `sumRange(left, right)`
- Return `pref[right + 1] - pref[left]`
- Example: nums = `[2, -1, 3, 5]`
  - pref = `[0, 2, 1, 4, 9]`
  - sumRange(1, 3) = pref[4] - pref[1] = 9 - 2 = 7
  - subarray `[-1, 3, 5]` sums to 7.

---

## Complexity (why this is good)
- Precompute prefix sums: **O(n)** time, **O(n)** memory
- Each query: **O(1)** time
- Perfect when: **array is immutable** (no updates) + **many sum queries**

---

## What to learn as a pattern (this is the key takeaway)

### Pattern name: “Prefix sum for fast range queries”
Whenever you see:
- lots of `sum(l..r)` queries
- “range sum”
- “running totals”
- “subarray sum”

Think:
1. Build `pref` with `pref[0]=0`
2. Use `pref[r+1]-pref[l]` for each query

This exact indexing (`n+1`, using `r+1`) is a very reusable trick because it:
- avoids `if (left == 0)` special case
- keeps formula consistent

---

## Common mistakes to avoid
1. **Off-by-one errors**
   - If `pref[i]` means sum up to index `i` inclusive, formulas get messy.
   - Easiest is: `pref[i] = sum of first i elements` (exclusive end).

2. **Forgetting `pref[0]=0`**
   - This causes incorrect sums for ranges starting at 0.

3. **Using `int` when sums can overflow**
   - In 303, constraints typically fit in `int`, but as a general habit:
     - use `long long` for prefix sums when values or n can be large.

Example safer version:

```cpp name=NumArray_longlong.cpp
#include <vector>
using namespace std;

class NumArray {
    vector<long long> pref;
public:
    NumArray(vector<int>& nums) {
        pref.assign(nums.size() + 1, 0);
        for (int i = 0; i < (int)nums.size(); i++) {
            pref[i+1] = pref[i] + nums[i];
        }
    }
    long long sumRange(int left, int right) {
        return pref[right+1] - pref[left];
    }
};
```
