For **703. Kth Largest Element in a Stream**, the core “heap pattern” you should learn in C++ is:

## What this problem teaches (the pattern)
### 1) “Kth largest so far” ⇒ keep only the **top K** elements
- If you keep the **largest K numbers seen so far**, then:
  - the **smallest** among those K numbers is the **kth largest overall**.
- So you want a structure where you can quickly:
  - insert a new number
  - remove the smallest (when you have more than K kept)
  - read the smallest among the kept K

That’s exactly a **min-heap of size K**.

In C++: `priority_queue<int, vector<int>, greater<int>> minHeap;`

---

## Step-by-step logic (streaming)
You are given `k` and an initial list `nums`. Then you’ll call `add(val)` many times.

### Step 0: Data structure choice
Use a **min-heap**:
- `top()` gives the **smallest** element in the heap
- heap size will never exceed `k`

### Step 1: Build the heap from initial numbers
For each `x` in `nums`:
1. `push(x)`
2. if `size() > k`, `pop()` (remove smallest)

After this:
- heap contains the **largest k** values from `nums`
- `heap.top()` is the **kth largest** among `nums` (if at least k elements exist; LeetCode’s constraints ensure calls are valid)

### Step 2: Process each new streamed value `val` in `add(val)`
1. `push(val)`
2. if `size() > k`, `pop()` (discard smallest)
3. return `heap.top()` (current kth largest)

That’s it.

---

## Why it works (key invariant to learn)
Maintain this invariant after every operation:

> **Invariant:** The heap contains the **largest K elements seen so far**.

- If you ever have more than K elements, removing the **smallest** is always safe:
  - because the smallest among (K+1) candidates cannot be in the top K.

Once invariant holds:
- the **kth largest** is the smallest among those top K ⇒ `minHeap.top()`.

## C++ implementation (idiomatic)
```cpp name=KthLargest.cpp
#include <bits/stdc++.h>
using namespace std;

class KthLargest {
    int k;
    priority_queue<int, vector<int>, greater<int>> pq; // min-heap
public:
    KthLargest(int k, vector<int>& nums) : k(k) {
        for (int x : nums) {
            pq.push(x);
            if ((int)pq.size() > k) pq.pop();
        }
    }

    int add(int val) {
        pq.push(val);
        if ((int)pq.size() > k) pq.pop();
        return pq.top();
    }
};
```

## What to learn specifically from this problem
### A) Recognizing the pattern prompt
Keywords that should trigger this exact approach:
- “**kth largest**”
- “**top k**”
- “in a **stream**” / “after each insertion”

### B) Choosing min-heap vs max-heap
- For **kth largest**, use **min-heap of size k** (most common).
- For **kth smallest**, use **max-heap of size k**.

### C) Complexity tradeoff you’re learning
Let `n` be number of items processed so far.
- Each `add`: `O(log k)` (not `O(log n)`)
- Memory: `O(k)`

This is the big win: **scale with k, not with stream size**.

### D) How to use C++ `priority_queue` with custom comparator
The syntax is a frequent stumbling block:
```cpp
priority_queue<int, vector<int>, greater<int>> pq;
```
This makes it a min-heap.

## Reusable heap templates (C++) for Top‑K / Kth / Closest patterns

### 0) Quick rule to choose heap type
- Want **K largest** / **kth largest** ⇒ keep **min-heap of size K**
- Want **K smallest** / **kth smallest** ⇒ keep **max-heap of size K**
- Want **K closest** (min distance) ⇒ keep **max-heap of size K** (so the *worst/ farthest* of the kept K is easy to remove)

---

## 1) Template: Kth Largest / Top K Largest (min-heap size K)

**Use for:**  
- “kth largest”, “top k largest”, streaming or array

**Idea:** keep only best K (largest). Root = smallest among them = kth largest.

```cpp name=top_k_largest_minheap_template.cpp
#include <bits/stdc++.h>
using namespace std;

// Returns the kth largest element in nums (k >= 1).
int kthLargest(vector<int>& nums, int k) {
    priority_queue<int, vector<int>, greater<int>> pq; // min-heap

    for (int x : nums) {
        pq.push(x);
        if ((int)pq.size() > k) pq.pop();              // drop smallest
    }
    return pq.top(); // kth largest
}

// Returns the top k largest elements (order not guaranteed).
vector<int> topKLargest(vector<int>& nums, int k) {
    priority_queue<int, vector<int>, greater<int>> pq; // min-heap

    for (int x : nums) {
        pq.push(x);
        if ((int)pq.size() > k) pq.pop();
    }

    vector<int> res;
    while (!pq.empty()) { res.push_back(pq.top()); pq.pop(); }
    // res currently is ascending among the kept elements; reverse if you want descending
    reverse(res.begin(), res.end());
    return res;
}
```

**What to memorize:**
- `greater<int>` makes it a min-heap
- Always: `push`, if `size>k` then `pop`
- Answer is `top()`

## 2) Template: Kth Smallest / Top K Smallest (max-heap size K)

**Use for:**  
- “kth smallest”, “top k smallest”

**Idea:** keep best K (smallest). Root = largest among them (the “worst” of the kept small ones) = kth smallest.

```cpp name=top_k_smallest_maxheap_template.cpp
#include <bits/stdc++.h>
using namespace std;

// Returns the kth smallest element.
int kthSmallest(vector<int>& nums, int k) {
    priority_queue<int> pq; // max-heap

    for (int x : nums) {
        pq.push(x);
        if ((int)pq.size() > k) pq.pop(); // drop largest
    }
    return pq.top(); // kth smallest
}

// Returns the top k smallest elements (order not guaranteed).
vector<int> topKSmallest(vector<int>& nums, int k) {
    priority_queue<int> pq; // max-heap

    for (int x : nums) {
        pq.push(x);
        if ((int)pq.size() > k) pq.pop();
    }

    vector<int> res;
    while (!pq.empty()) { res.push_back(pq.top()); pq.pop(); }
    // res currently is descending among kept; reverse for ascending
    reverse(res.begin(), res.end());
    return res;
}
```

**What to memorize:**
- default `priority_queue<int>` is max-heap
- same “push then trim to size k”

## 3) Template: K Closest Points / K smallest by a key (max-heap size K)

**Use for:**  
- “k closest”, “k nearest”, “k smallest distance”, “top k by score/key”
- where “best” means *smallest metric*

**Idea:** keep K best (smallest distance). To evict quickly, store them in a **max-heap by distance**, so the farthest among kept is on top.

```cpp name=k_closest_points_template.cpp
#include <bits/stdc++.h>
using namespace std;

// Example: LeetCode 973 K Closest Points to Origin
vector<vector<int>> kClosest(vector<vector<int>>& points, int k) {
    // (dist2, index) in max-heap (largest dist2 on top)
    priority_queue<pair<long long,int>> pq;

    for (int i = 0; i < (int)points.size(); i++) {
        long long x = points[i][0], y = points[i][1];
        long long dist2 = x*x + y*y;

        pq.push({dist2, i});
        if ((int)pq.size() > k) pq.pop(); // remove farthest so far
    }

    vector<vector<int>> res;
    while (!pq.empty()) {
        res.push_back(points[pq.top().second]);
        pq.pop();
    }
    return res; // order not guaranteed
}
```

**What to memorize:**
- For “closest/smallest metric”: use **max-heap** so you can pop the worst (largest metric).
- Store `(metric, payload)` as a pair.

## 4) Template: Merge K Sorted Lists / Arrays (min-heap of current heads)

**Use for:**  
- “merge k sorted”, “smallest range covering k lists”, “k-way merge”
- You repeatedly take the smallest “current head” among k sources.

```cpp name=merge_k_sorted_arrays_template.cpp
#include <bits/stdc++.h>
using namespace std;

struct Node {
    int val;
    int arr;  // which array
    int idx;  // index inside that array
};

struct Cmp {
    bool operator()(const Node& a, const Node& b) const {
        return a.val > b.val; // min-heap by val
    }
};

vector<int> mergeKSortedArrays(const vector<vector<int>>& a) {
    priority_queue<Node, vector<Node>, Cmp> pq;

    // initialize with first element of each array
    for (int i = 0; i < (int)a.size(); i++) {
        if (!a[i].empty()) pq.push({a[i][0], i, 0});
    }

    vector<int> out;
    while (!pq.empty()) {
        Node cur = pq.top(); pq.pop();
        out.push_back(cur.val);

        int nextIdx = cur.idx + 1;
        if (nextIdx < (int)a[cur.arr].size()) {
            pq.push({a[cur.arr][nextIdx], cur.arr, nextIdx});
        }
    }
    return out;
}
```

**What to memorize:**
- Heap size is at most `k` (number of lists)
- Always push the next element from the same list you popped from
- Time: `O(N log k)` where N is total elements
