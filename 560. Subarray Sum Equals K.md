Prefix sum + hashmap for **“count subarrays with sum = K”** is the go-to pattern when you need **number of subarrays** (not longest) with a given sum and the array can have **negative numbers** (so sliding window won’t work reliably).

---

## Core idea (the pattern)

### 1) Prefix sum definition
Let:

- `prefix[i] = nums[0] + nums[1] + ... + nums[i]`

Sum of subarray `nums[l..r]` is:

- `prefix[r] - prefix[l-1]`

We want:

- `prefix[r] - prefix[l-1] = k`
- so `prefix[l-1] = prefix[r] - k`

Meaning:
> For each index `r`, if we know how many times the value `(prefix[r] - k)` occurred as a previous prefix sum, we can add that count to the answer.

### 2) Hashmap stores counts of prefix sums
Maintain a map:

- `count[p] = how many times prefix sum p has occurred so far`

When you compute current prefix sum `sum`:
- number of valid subarrays ending at current index = `count[sum - k]`
- then update: `count[sum]++`

### 3) Critical initialization
Set:
- `count[0] = 1`

Why?
- It handles subarrays starting at index 0.
- Example: if `sum == k` at some index, then `sum - k == 0`, and we want to count that as 1 valid subarray.

---

## Walkthrough example
`nums = [1, 2, 3], k = 3`

Initialize:
- `count = {0:1}`
- `sum = 0`, `ans = 0`

Index 0: add 1 → `sum=1`
- look for `sum-k = -2` → not found
- `count[1]++`

Index 1: add 2 → `sum=3`
- look for `sum-k = 0` → found 1 → `ans += 1` (subarray [1,2])
- `count[3]++`

Index 2: add 3 → `sum=6`
- look for `sum-k = 3` → found 1 → `ans += 1` (subarray [3])
- `count[6]++`

Answer = 2.

## C++ implementation (standard pattern)

```cpp name=subarray_sum_equals_k.cpp
#include <bits/stdc++.h>
using namespace std;

class Solution {
public:
    int subarraySum(vector<int>& nums, int k) {
        unordered_map<int, int> cnt;  // prefixSum -> frequency
        cnt[0] = 1;                   // empty prefix sum occurs once

        int sum = 0;
        int ans = 0;

        for (int x : nums) {
            sum += x;

            // count how many previous prefix sums were (sum - k)
            if (cnt.find(sum - k) != cnt.end()) {
                ans += cnt[sum - k];
            }

            // record current prefix sum
            cnt[sum]++;
        }

        return ans;
    }
};
```

### Complexity
- Time: **O(n)** average
- Space: **O(n)** for hashmap

## What to “learn” as a reusable template

### When this pattern applies
Use **prefix sum + hashmap counts** when:
- You need **count of subarrays** with sum = K
- Array can contain **negative / zero / positive**
- Similar problems ask for:
  - count of subarrays with sum divisible by K
  - count of subarrays with equal number of 0s and 1s
  - count of subarrays with XOR = K (same pattern, but prefix XOR)

### The invariant to memorize
After processing elements up to index `i`, hashmap `cnt` stores:
- frequencies of all prefix sums `prefix[0..i]` (and also 0 once initially)

At index `i`, number of valid subarrays ending at `i` is:
- `cnt[prefix[i] - k]`

---

## Common mistakes (worth remembering)
1) **Forgetting `cnt[0] = 1`** → misses subarrays starting at index 0  
2) Updating `cnt[sum]++` **before** counting → can overcount when `k=0`  
   - Correct order: **count first**, then **insert**
3) Using sliding window even with negatives → wrong, because window sum isn’t monotonic.
