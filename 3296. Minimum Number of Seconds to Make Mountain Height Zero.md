### How “Binary search on answer + feasibility check” applies to **LeetCode 3296 — Minimum Number of Seconds to Make Mountain Height Zero**

This problem is a textbook **parametric search** problem:

- You’re asked for the **minimum time `T`** (seconds) such that **it is possible** to reduce the mountain height to zero within `T`.
- If it’s possible in `T` seconds, then it��s also possible in **any larger time** `T+1, T+2, ...`  
  That **monotonic (True/False) behavior** is the trigger for **binary search on the answer**.

---

## 1) Recognizing the pattern (how to spot it quickly)

Look for these signals:

### Signal A: “minimum / earliest / smallest X such that …”
The title itself: **Minimum Number of Seconds**.

### Signal B: You can “simulate” or “check” for a fixed candidate answer
Ask:  
> “If I *pretend* I have `T` seconds, can I finish the task?”

If you can write a function like:

- `canFinish(T) -> boolean`

then you’re halfway to this pattern.

### Signal C: Monotonic feasibility
If `canFinish(T)` is true, then `canFinish(T+1)` must be true.

That gives you a sorted boolean array over time:

```
T:   0 1 2 3 4 5 6 ...
can: F F F T T T T ...
```

Binary search finds the first `T` where it becomes true.

---

## 2) What is the “feasibility check” in this problem?

In this problem, you have workers with some speed/rate behavior (the exact mechanics vary, but the key idea is consistent):

For a given time `T`, each worker can contribute some amount of “height reduction” (or “work units”) by time `T`.

So feasibility is:

> Compute total reducible height by all workers within `T`.  
> If total ≥ mountain height `H`, then `T` is feasible.

So:

- **Answer space**: time `T`
- **Check**: sum of each worker’s maximum contribution within `T`

---

## 3) Why the check is typically computed with another binary search / math

In many versions of this problem, a worker’s work over time is not linear like `rate * T`; it’s something like:

- the `k`-th unit of work takes `k * workerTime` seconds (increasing cost)
- or some arithmetic progression

That means for one worker:

- time needed to do `k` units is something like  
  `workerTime * (1 + 2 + ... + k) = workerTime * k*(k+1)/2`

Then in the feasibility check for a fixed `T`, you want max `k` such that:

```
workerTime * k*(k+1)/2 <= T
```

That is monotonic in `k`, so you can find `k` by:
- solving quadratic formula, or
- binary searching `k`

So the overall structure often becomes:

- **Outer binary search** over `T` (answer)
- **Inner per-worker computation** of max `k` achievable by time `T`

---

## 4) Step-by-step learning approach (mental model)

### Step 1: Convert “min time” to “decision problem”
Instead of “find min time”, ask:

> “Given `T`, is it possible?”

### Step 2: Prove monotonicity
Convince yourself:

- more time cannot make it worse
- therefore feasibility is monotone

### Step 3: Write `can(T)`
Inside it:
- for each worker, compute how much they can do in `T`
- sum it up
- compare to `H`

### Step 4: Binary search the smallest feasible `T`
Standard “first true” binary search.

## 5) Reusable code pattern (template)

### A) Outer binary search: “first true”
This is reusable across tons of problems.

```python name=parametric_search_template.py
def first_true(lo, hi, can):
    """
    Find smallest x in [lo, hi] such that can(x) is True.
    Assumes can is monotone: False ... False True ... True
    """
    ans = hi
    while lo <= hi:
        mid = (lo + hi) // 2
        if can(mid):
            ans = mid
            hi = mid - 1
        else:
            lo = mid + 1
    return ans
```

### B) Feasibility check skeleton for this problem

```python name=feasibility_skeleton.py
def can(T, H, workers):
    total = 0
    for w in workers:
        # compute max units k worker can do within time T
        k = max_units_in_time(T, w)
        total += k
        if total >= H:
            return True
    return total >= H
```

### C) “max units in time” (when time per unit increases like 1+2+...+k)
If a worker needs `w * k*(k+1)/2 <= T`, you can compute k.

**Option 1: binary search k** (simple + safe):

```python name=max_units_bs.py
def max_units_in_time(T, w):
    lo, hi = 0, 10**6  # hi should be chosen based on constraints
    while lo < hi:
        mid = (lo + hi + 1) // 2
        if w * mid * (mid + 1) // 2 <= T:
            lo = mid
        else:
            hi = mid - 1
    return lo
```

**Option 2: quadratic formula** (faster, but more error-prone):
Solve `w*k*(k+1)/2 <= T`.

## 6) How to choose search bounds for time `T`

Typical approach:

- `lo = 0` (or 1)
- `hi` big enough that it’s definitely possible

Ways to find `hi`:
- Start with `hi = 1` and **double** until `can(hi)` becomes true (very reusable technique).
- Or derive a safe upper bound from constraints.

Example doubling pattern:

```python name=upper_bound_doubling.py
hi = 1
while not can(hi, H, workers):
    hi *= 2
lo = 0
answer = first_true(lo, hi, lambda T: can(T, H, workers))
```

This avoids messing up with constraint math.

---

## 7) Common mistakes to watch for

1) **Feasibility not monotone**  
If extra time could somehow reduce capability (rare), binary search fails.

2) **Overflow**  
In checks like `w * k*(k+1)/2`, use 64-bit (`long long` in C++ / Python int is fine).

3) **Too-slow feasibility**  
Outer BS is `O(log hi)`.  
Make `can(T)` efficient, ideally `O(n)` with O(1) per worker (math), or `O(n log something)` if needed.

### The key check to invent

> If I fix `T`, how much “height” can all workers reduce by time `T`?

## 2) Feasibility check for LC 3296 (core math)

In this problem, each worker has a parameter `t` (time base).  
To reduce `k` units of height, that worker needs:

\[
t \cdot (1 + 2 + ... + k) = t \cdot \frac{k(k+1)}{2}
\]

So for a fixed `T`, that worker can do the **largest `k`** such that:

\[
t \cdot \frac{k(k+1)}{2} \le T
\]

That’s monotone in `k`, so you can compute `k` via:
- another binary search on `k`, or
- quadratic formula

In C++ interviews/contests, the **binary search on `k`** is easiest to implement safely.

## 3) Reusable C++ code pattern (template)

### A) “first true” binary search on answer
This template is reusable in many problems:

```cpp name=parametric_search_template.cpp
template <class F>
long long first_true(long long lo, long long hi, F can) {
    // returns smallest x in [lo, hi] with can(x) == true
    while (lo < hi) {
        long long mid = lo + (hi - lo) / 2;
        if (can(mid)) hi = mid;
        else lo = mid + 1;
    }
    return lo;
}
```

## 4) Implement `can(T)` in C++ (feasibility check)

### B) Max units a single worker can do in time `T`
Binary search `k`:

```cpp name=max_units_in_time.cpp
static long long maxUnitsInTime(long long T, long long t) {
    // Find max k such that t * k*(k+1)/2 <= T
    long long lo = 0, hi = 1;

    // Exponentially grow hi until condition fails (safe bound)
    auto ok = [&](long long k) -> bool {
        __int128 lhs = (__int128)t * k * (k + 1) / 2;
        return lhs <= T;
    };

    while (ok(hi)) {
        hi *= 2;
        // Prevent runaway; though __int128 makes comparison safe,
        // we can stop if hi gets huge.
        if (hi > (long long)2e9) break;
    }

    // Now binary search in [lo, hi]
    while (lo < hi) {
        long long mid = lo + (hi - lo + 1) / 2;
        if (ok(mid)) lo = mid;
        else hi = mid - 1;
    }
    return lo;
}
```

Key C++ detail: use `__int128` to avoid overflow in `t * k * (k+1)`.

### C) Feasibility check across all workers
Sum their contributions and early-exit once you reach `H`:

```cpp name=can_function.cpp
static bool canFinish(long long T, long long H, const std::vector<int>& workerTimes) {
    long long total = 0;
    for (int t : workerTimes) {
        total += maxUnitsInTime(T, t);
        if (total >= H) return true;
    }
    return total >= H;
}
```

## 6) Full C++ solution skeleton (put together)

```cpp name=lc3296_parametric_search.cpp
#include <bits/stdc++.h>
using namespace std;

class Solution {
    static long long maxUnitsInTime(long long T, long long t) {
        auto ok = [&](long long k) -> bool {
            __int128 lhs = (__int128)t * k * (k + 1) / 2;
            return lhs <= T;
        };

        long long lo = 0, hi = 1;
        while (ok(hi)) {
            hi *= 2;
            if (hi > (long long)2e9) break;
        }

        while (lo < hi) {
            long long mid = lo + (hi - lo + 1) / 2;
            if (ok(mid)) lo = mid;
            else hi = mid - 1;
        }
        return lo;
    }

    static bool canFinish(long long T, long long H, const vector<int>& workerTimes) {
        long long total = 0;
        for (int t : workerTimes) {
            total += maxUnitsInTime(T, t);
            if (total >= H) return true;
        }
        return total >= H;
    }

public:
    long long minNumberOfSeconds(int mountainHeight, vector<int>& workerTimes) {
        long long H = mountainHeight;

        long long lo = 0, hi = 1;
        while (!canFinish(hi, H, workerTimes)) hi *= 2;

        while (lo < hi) {
            long long mid = lo + (hi - lo) / 2;
            if (canFinish(mid, H, workerTimes)) hi = mid;
            else lo = mid + 1;
        }
        return lo;
    }
};
```
