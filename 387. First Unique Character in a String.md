* It’s not “index,count”. It’s character,count.
Key should be char (or unsigned char), not “index”.

Use char c : s (or unsigned char c : s). Using int works but is confusing.
Second loop: return i;

* i is the character value (like 'a'), not the index.
The problem asks for the index of the first unique character.
So you must iterate with an index for the second pass.

In a hash map, the **key must identify what you’re counting.** In this problem you need:

frequency of each **character** in the whole string
You don’t need:

frequency of each **index** (each index occurs exactly once)
frequency of “character at index i” (that’s just the character itself)
Why “count of char at that index” doesn’t make sense as a map key
Index i is unique in the string. So if you keyed by index:

```cnt[i]++```

then every cnt[i] would become 1 (because you visit each index once). That cannot tell you duplicates.

Example: s = "leetcode"
* index 0 ('l'): cnt[0]=1
* index 1 ('e'): cnt[1]=1
* index 2 ('e'): cnt[2]=1 ← still 1, but 'e' is duplicated So “index → count” loses the key information you need (the character).

## What you actually need for 387
You need to answer: “Is s[i] unique in the entire string?”

That requires:

* freq['l'] = 1
* freq['e'] = 3
* freq['t'] = 1 etc.
So the correct mapping is:

* **key = character**
* **value = frequency**

```cpp
unordered_map<char,int> freq;
for (char c : s) freq[c]++; // counts per character, across all indices
```
Then checking:
```cpp
if (freq[s[i]] == 1) return i;
```
## When would “index → something” be useful?
Index-based maps/arrays are used when the state is about positions, e.g.:

* DP (dp[i])
* prefix sums (pref[i])
* “next greater element index” (next[i]) But for frequency, indices are the wrong identity; duplicates happen across indices.
## Small note about your original loop
You wrote ```for (int i : s) cnt[i]++;```
Even though ```i``` is an ```int```, it still represents the character code (like 97 for 'a'), not a string index. So it kind of works as “character → count”, but it’s confusing and then the second pass returns the wrong thing (you returned ```i``` which is the character code, not the index).

# Clean correct solution (frequency map approach)
Since LeetCode 387 is lowercase letters, the best is array<int,26>:
```cpp
#include <bits/stdc++.h>
using namespace std;

class Solution {
public:
    int firstUniqChar(string s) {
        array<int, 26> cnt{};
        for (char c : s) cnt[c - 'a']++;

        for (int i = 0; i < (int)s.size(); i++) {
            if (cnt[s[i] - 'a'] == 1) return i;
        }
        return -1;
    }
};
```
## Invariant/state (what you’re doing)
* After first pass: cnt[ch] = frequency of each character in s
* Second pass: first position i such that cnt[s[i]] == 1 is your answer
