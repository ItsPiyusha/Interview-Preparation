Reusable pattern to learn for **704. Binary Search** is the **“classic binary search on indices”** (a.k.a. `while (lo <= hi)`), which returns as soon as it finds the target.

### Step-by-step (what to memorize)
1) **Initialize bounds**
- `lo = 0`
- `hi = n - 1`

2) **Loop while search space is valid**
- Condition: `while (lo <= hi)`  
  Meaning: there is still at least one candidate index.

3) **Pick middle safely**
- `mid = lo + (hi - lo) / 2` (avoids overflow)

4) **Compare and shrink**
- If `nums[mid] == target` → return `mid`
- If `nums[mid] < target` → target must be on the right → `lo = mid + 1`
- Else (`nums[mid] > target`) → target must be on the left → `hi = mid - 1`

5) **Not found**
- Return `-1`

---

### C++ code (reusable template)
```cpp name=leetcode_704_binary_search.cpp
#include <vector>
using namespace std;

class Solution {
public:
    int search(vector<int>& nums, int target) {
        int lo = 0;
        int hi = (int)nums.size() - 1;

        while (lo <= hi) {
            int mid = lo + (hi - lo) / 2;

            if (nums[mid] == target) {
                return mid;
            } else if (nums[mid] < target) {
                lo = mid + 1;      // discard left half including mid
            } else {
                hi = mid - 1;      // discard right half including mid
            }
        }
        return -1;
    }
};
```

---

### The invariant (why it works, reusable idea)
At every moment, the target (if it exists) must be inside the **current search interval**:
- `nums[lo..hi]` is the remaining candidate range.
- Each comparison removes half the range without losing the target.
