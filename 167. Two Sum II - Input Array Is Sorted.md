### Problem (LeetCode 167: Two Sum II – Input Array Is Sorted)
You are given a **sorted** array `numbers` (non-decreasing order) and an integer `target`.  
Find **two distinct elements** such that they add up to `target`, and return their indices as **1-based** indices.

There is **exactly one solution**, and you may not use the same element twice.

---

## Why “two pointers from opposite ends” works
Because the array is sorted, moving pointers changes the sum in a predictable way.

Let:
- `l` = left pointer (starts at the beginning, smallest values)
- `r` = right pointer (starts at the end, largest values)

At any time, consider:
`sum = numbers[l] + numbers[r]`

### Case 1: `sum == target`
We found the pair. Return `{l+1, r+1}` (because the problem wants 1-based indices).

### Case 2: `sum < target`  (sum is too small)
We need a **bigger** sum.
- If we decrease `r`, we’d pick a smaller number on the right → sum would become even smaller or not increase.
- The only way to possibly increase the sum is to move `l` to the right (`l++`), because `numbers[l]` will get larger (or stay same).

So: **`l++`**

### Case 3: `sum > target`  (sum is too big)
We need a **smaller** sum.
- If we increase `l`, we’d pick a bigger number on the left → sum would become even bigger or not decrease.
- The correct move is to move `r` left (`r--`), because `numbers[r]` will get smaller (or stay same).

So: **`r--`**

This logic works *only* because the array is sorted.

---

## Key invariant (what stays true as we move pointers)
At any point, all pairs that use:
- indices `< l` with current `r` are already too small (we moved `l` past them because sums were too small)
- indices `> r` with current `l` are already too large (we moved `r` past them because sums were too large)

So we never “miss” the correct pair—each move eliminates a whole set of impossible pairs.

---

## Step-by-step example (dry run)
Example:  
`numbers = [2, 7, 11, 15]`, `target = 9`

- `l=0 (2)`, `r=3 (15)` → sum = 17 (too big) → `r--`
- `l=0 (2)`, `r=2 (11)` → sum = 13 (too big) → `r--`
- `l=0 (2)`, `r=1 (7)` → sum = 9 (match) → return `{1, 2}`

Only 3 comparisons, and we’re done.

---

## Complexity
- **Time: O(n)**  
  Each pointer only moves inward. `l` can move at most `n-1` times and `r` can move at most `n-1` times, but total moves are bounded by `n`.
- **Space: O(1)**  
  Only a few variables used.

---

## C++ Code
```cpp name=two_sum_ii.cpp
#include <vector>
using namespace std;

class Solution {
public:
    vector<int> twoSum(vector<int>& numbers, int target) {
        int l = 0, r = (int)numbers.size() - 1;

        while (l < r) {
            long long sum = (long long)numbers[l] + numbers[r];

            if (sum == target) {
                return {l + 1, r + 1};   // 1-based indexing
            } else if (sum < target) {
                ++l;                     // increase sum
            } else {
                --r;                     // decrease sum
            }
        }
        return {}; // guaranteed to have exactly one solution
    }
};
```
