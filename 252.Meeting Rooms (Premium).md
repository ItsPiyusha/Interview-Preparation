### LeetCode 252 — Meeting Rooms (Premium)

#### Problem statement
You are given an array of meeting time intervals `intervals`, where each interval is `[start, end]`.  
Return `true` if a person can attend **all** meetings (i.e., **no overlaps**), otherwise return `false`.

- Meetings that “touch” are OK: if one ends exactly when another starts, e.g. `[0, 10]` and `[10, 15]`, that is **not** an overlap.

---

### Example
**Input:** `[[0,30],[5,10],[15,20]]`  
Sort by start ⇒ `[[0,30],[5,10],[15,20]]`  
- Compare `[0,30]` with `[5,10]`: since `5 < 30`, they overlap ⇒ **return false**

**Output:** `false`

Another quick one:

**Input:** `[[7,10],[2,4]]`  
Sort ⇒ `[[2,4],[7,10]]`  
- Compare `[2,4]` with `[7,10]`: `7 < 4`? no ⇒ no overlap ⇒ **true**

**Output:** `true`

---

## Pattern to learn: “Intervals feasibility via sorting”
**Use when:** you need to determine if intervals can be scheduled without overlap (or select non-overlapping ones).

### Key idea
If you **sort by start time**, then you only need to check each meeting against the **previous meeting’s end**.

Why this works:
- After sorting, the only interval that can conflict with the current one *the earliest* is the one that ends last among previous intervals—maintained by tracking `prevEnd`.
- For a simple “any overlap?” check, comparing with the immediate previous interval is enough after sorting by start.

---

## Line-by-line approach (what each line is teaching you)

### Step 1: Sort the intervals by start time
**What you learn:** Sorting turns a global overlap question into a simple neighbor check.

Pseudo:
- `sort intervals by intervals[i][0]` (start)

After sorting:
- if an overlap exists anywhere, it will appear between some adjacent intervals in this sorted order.

---

### Step 2: Track the end time of the last meeting you accepted
Initialize:
- `prevEnd = end time of the first interval`

**What you learn:** Greedy tracking—keep the last finishing boundary you must respect.

---

### Step 3: Scan from the 2nd meeting onward
For each interval `[start, end]`:

#### Overlap condition
If:
- `start < prevEnd`  
then meetings overlap ⇒ return `false`

**What you learn:** In interval problems, overlap check is usually:
- overlap exists if next.start < current.end  
- (and non-overlap if next.start >= current.end)

#### Otherwise update
- `prevEnd = end`

**What you learn:** If you can attend this meeting, your new “busy until” time becomes its end.

---

### Step 4: If you never found overlap, return true
**What you learn:** Feasibility checks often exit early on the first violation.

---

## Reference implementation (Python) with “teaching comments”
```python name=meeting_rooms_252.py
from typing import List

class Solution:
    def canAttendMeetings(self, intervals: List[List[int]]) -> bool:
        # 1) Sort by start time so we only need to check neighbors.
        intervals.sort(key=lambda x: x[0])

        # 2) Walk through and ensure no meeting starts before the previous one ends.
        for i in range(1, len(intervals)):
            prev_end = intervals[i - 1][1]
            curr_start = intervals[i][0]

            # 3) Overlap check (strictly less => overlap).
            if curr_start < prev_end:
                return False

        # 4) No overlaps found.
        return True
```

### Same idea, slightly more “greedy boundary” style
```python name=meeting_rooms_252_boundary_style.py
from typing import List

class Solution:
    def canAttendMeetings(self, intervals: List[List[int]]) -> bool:
        intervals.sort(key=lambda it: it[0])

        prevEnd = float("-inf")
        for start, end in intervals:
            if start < prevEnd:
                return False
            prevEnd = end

        return True
```

---

## Complexity
- Sorting: `O(n log n)`
- Scan: `O(n)`
- Extra space: `O(1)` (ignoring sort’s internal space)

---

## What to memorize for this pattern
1. **Sort** intervals (usually by **start**; sometimes by **end** depending on goal).
2. Maintain a **boundary** (`prevEnd`).
3. Overlap check is typically: `next.start < prevEnd`
4. Update boundary: `prevEnd = next.end`

If you want, I can also show the “select maximum number of non-overlapping intervals” variant (sort by **end**, greedy pick) since that’s the other half of the classic intervals-greedy pattern.

```cpp name=meeting_rooms_252.cpp
#include <bits/stdc++.h>
using namespace std;

class Solution {
public:
    bool canAttendMeetings(vector<vector<int>>& intervals) {
        // 1) Sort meetings by start time (and by end time as a tie-breaker)
        sort(intervals.begin(), intervals.end(),
             [](const vector<int>& a, const vector<int>& b) {
                 if (a[0] != b[0]) return a[0] < b[0];
                 return a[1] < b[1];
             });

        // 2) After sorting, only adjacent intervals can overlap.
        //    Scan and check if current meeting starts before previous ends.
        for (int i = 1; i < (int)intervals.size(); i++) {
            int prevEnd = intervals[i - 1][1];
            int currStart = intervals[i][0];

            // 3) Overlap condition: currStart < prevEnd
            //    (If currStart == prevEnd, it's fine: back-to-back meetings.)
            if (currStart < prevEnd) return false;
        }

        // 4) No overlap found
        return true;
    }
};
```
